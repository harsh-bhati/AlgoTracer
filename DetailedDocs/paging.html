What is Paging?

Paging is a memory management scheme that eliminates the need for contiguous allocation of physical memory. It allows the physical address space of a process to be non-contiguous, which helps to avoid external fragmentation and makes memory allocation more flexible.

Key Concepts

  • Page: A fixed-size block of logical memory. The process’s logical memory is divided into pages of equal size.
  • Frame: A fixed-size block of physical memory. Physical memory is divided into frames of the same size as pages.
  • Page Table: A data structure used to map logical pages to physical frames.

How Paging Works

  1. The process’s logical address space is divided into pages.
  2. Physical memory is divided into frames of the same size as pages.
  3. When a process is executed, its pages are loaded into available frames in physical memory.
  4. The page table keeps track of where each page is stored in physical memory.
  5. Logical addresses generated by the CPU are translated into physical addresses using the page table.

Advantages of Paging

  • Eliminates external fragmentation by allowing non-contiguous allocation of physical memory.
  • Allows processes to be larger than physical memory by loading pages as needed.
  • Simplifies memory management and increases utilization.

Disadvantages of Paging

  • Internal fragmentation may occur if a process does not fully utilize the last page.
  • Page table overhead can be large, especially for processes with large address spaces.
  • Address translation requires additional memory access, which can slow down execution (mitigated by Translation Lookaside Buffers).
